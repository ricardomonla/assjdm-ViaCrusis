## Objetivo

Implementar un mecanismo de deploy y migración de datos **sin acceso SSH/SCP** al servidor de producción. Usar el webhook de GitHub para ejecutar migraciones de DB automáticamente tras cada push.

**Problema resuelto:** No hay acceso físico ni remoto (SSH/SCP) al servidor de producción. La DB vive en el servidor y no se puede modificar directamente.

**Solución:** Los datos se exportan a JSON, se commitean al repo, y el webhook los importa automáticamente en producción.

## Arquitectura

### Flujo de Deploy Automático

```
┌──────────────┐      ┌─────────────────┐      ┌──────────────────┐
│   GitHub     │ ───→ │  deploy.php     │ ───→ │  Producción      │
│   (push)     │      │  (webhook)      │      │  (rmonla.duckdns.org) │
└──────────────┘      └─────────────────┘      └──────────────────┘
                              │
                              ▼
                       data/db_import.php
                       (migración DB)
```

### Flujo de Migración de Datos

```
┌─────────────────┐
│ 1. Test (Docker)│
│    DB SQLite    │
└────────┬────────┘
         │ php tools/db_export.php
         ▼
┌─────────────────┐
│ 2. JSON         │
│    migration_   │
│    personas.json│
└────────┬────────┘
         │ git add + commit + push
         ▼
┌─────────────────┐
│ 3. GitHub       │
│    (repo)       │
└────────┬────────┘
         │ webhook POST
         ▼
┌─────────────────┐
│ 4. Producción   │
│    deploy.php   │
│    ↓            │
│    db_import.php│
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 5. DB Producción│
│    actualizada  │
└─────────────────┘
```

## Features

### Feature 1: Webhook de Deploy (deploy.php)

| Aspecto | Detalle |
|---------|---------|
| URL | `https://rmonla.duckdns.org/vcby/deploy.php` |
| Trigger | Push a rama `main` en GitHub |
| Acción | `git pull origin main` |
| Secret | `vcby2026deploy` |
| Log | `deploy.log` |

**Código:**
```php
// deploy.php
exec('git pull origin main 2>&1', $output, $returnCode);
if ($returnCode === 0) {
    exec('php data/db_import.php 2>&1', $migrateOutput, $migrateCode);
}
```

### Feature 2: Exportar DB Local (tools/db_export.php)

| Aspecto | Detalle |
|---------|---------|
| Input | DB SQLite local (`tools/test-server/vcby-data/vcby.db`) |
| Output | `data/migration_personas.json` |
| Tablas | `roles`, `personas`, `persona_roles` |
| Formato | JSON con version, date, tables |

**Comando:**
```bash
docker exec vcby-test php /app/tools/db_export.php
```

### Feature 3: Importar DB en Producción (data/db_import.php)

| Aspecto | Detalle |
|---------|---------|
| Input | `data/migration_personas.json` (traído por git pull) |
| Output | DB SQLite en producción |
| Método | `INSERT OR IGNORE` (idempotente) |
| Log | `data/import.log` |
| Post-import | Renombra JSON a `.imported` (evita re-import) |

### Feature 4: Endpoint de Verificación (status API)

| Aspecto | Detalle |
|---------|---------|
| URL | `/personas/api.php?action=status` |
| Método | GET |
| Respuesta | JSON con counts, migration status, timestamp |

**Respuesta esperada:**
```json
{
  "ok": true,
  "status": "healthy",
  "version": "26.12",
  "counts": {
    "personas": 27,
    "roles": 6,
    "persona_roles": 27
  },
  "migration": {
    "done": true,
    "has_real_data": true
  },
  "last_person_created": "2026-04-18 00:22:02",
  "timestamp": "2026-04-18 01:00:00"
}
```

### Feature 5: Configuración Webhook en GitHub

**En GitHub:**
1. Ir a `Settings → Webhooks → Add webhook`
2. Configurar:
   - **Payload URL:** `https://rmonla.duckdns.org/vcby/deploy.php`
   - **Content type:** `application/json`
   - **Secret:** `vcby2026deploy`
   - **Events:** `Just the push event`

## Troubleshooting

### El webhook no dispara

1. Verificar en GitHub: `Settings → Webhooks → Recent Deliveries` — ¿último delivery fue 200 OK?
2. Verificar `deploy.log` en producción
3. Verificar que `deploy.php` es accesible: `curl https://rmonla.duckdns.org/vcby/deploy.php`

### La migración falla

1. Verificar que `data/db_import.php` existe en producción
2. Verificar que el JSON es válido:
   ```bash
   php -r "json_decode(file_get_contents('data/migration_personas.json')); echo json_last_error_msg();"
   ```
3. Verificar permisos de escritura en `data/`

### Datos duplicados

El script usa `INSERT OR IGNORE`, así que:
- Personas con mismo `(nombre, apellido)` se saltan
- Roles con mismo `id` se saltan
- Persona_roles duplicados se saltan

Si necesitas re-importar, borrá los datos primero o eliminá el archivo `.imported` para reintentar.

## Próximas Migraciones

Para migrar datos de **nuevas tablas** en el futuro:

### 1. Agregar tabla a `db_export.php`
```php
$export['tables']['nueva_tabla'] = $db->query("SELECT * FROM nueva_tabla")->fetchAll(PDO::FETCH_ASSOC);
```

### 2. Agregar import a `db_import.php`
```php
if (!empty($export['tables']['nueva_tabla'])) {
    $stmt = $db->prepare("INSERT OR IGNORE INTO nueva_tabla (col1, col2, ...) VALUES (?, ?, ...)");
    foreach ($export['tables']['nueva_tabla'] as $row) {
        $stmt->execute([$row['col1'], $row['col2'], ...]);
    }
    $imported['nueva_tabla'] = count($export['tables']['nueva_tabla']);
}
```

### 3. Exportar, commitear, pushear
```bash
docker exec vcby-test php /app/tools/db_export.php
git add data/migration_personas.json
git commit -m "v26.X: Migración DB - nueva_tabla"
git push origin main
```
