## Objetivo

1. **Fade-Out per-track:** Switch configurable por el Director (SQLite) que aplica fade-out antes de navegar.
2. **Audio Preload:** Precargar el siguiente audio para transición imperceptible.
3. **Selector de Escena:** Combo/select para saltar entre audios del mismo grupo.
4. **Entorno de Test Docker:** Réplica fiel de producción con DB real.

### Feature 5: Reorganización del Versionado ✅

**Problema:** El historial tiene entradas excesivas por mini-cambios que deberían agruparse en una sola versión mayor.

**Estado actual:**

| Rango | Cantidad | Problema |
|-------|----------|----------|
| `5.4.1` → `5.6` | 8 entradas | Legacy 2024-2025, OK |
| `25.1` → `25.5` | 5 entradas | Limpias, OK |
| `26.1` → `26.7` | 7 entradas | Limpias, OK |
| `26.8.1` → `26.8.29` | **29 entradas** | ⚠ Excesivo — consolidar en `26.8` |
| `26.9.1` → `26.9.2` | 2 entradas | Consolidar en `26.9` |
| `26.10.1` → `26.10.8` | **8 entradas** | ⚠ Consolidar en `26.10` |

**Propuesta:** Consolidar sub-versiones en una sola entrada por versión mayor:

| Antes | Después | Entradas |
|-------|---------|----------|
| `26.8.1` a `26.8.29` | `26.8` | 29 → 1 |
| `26.9.1` a `26.9.2` | `26.9` | 2 → 1 |
| `26.10.1` a `26.10.8` | `26.10` | 8 → 1 |
| Nueva | `26.11` | Esta versión |

Cada entrada consolidada agrupa los cambios más importantes y usa la fecha del último sub-release.

**Criterio nuevo:** Una versión = un release testeado y desplegado. No más sub-versiones por hotfixes intermedios.

**Mecánica de versionado (se documenta como header en `versionLogs.php`):**

```
Formato:  AÑ.NN  (año 2 dígitos . número secuencial)
Ejemplos: 26.11, 26.12, 27.1

Reglas:
  1. Cada versión es un release completo: planificado, implementado,
     testeado en Docker y verificado en producción.
  2. No se crean sub-versiones (X.Y.Z). Si hay un bug post-deploy,
     se abre una nueva versión (X.Y+1) con su propio ciclo.
  3. El bump se hace SOLO en Fase 4 (Deploy), después de pasar
     todos los tests de Fase 3.

Flujo de trabajo:
  ┌─────────────┐   ┌──────────────┐   ┌──────────────┐
  │ Planificar  │ → │ Implementar  │ → │ Testear      │
  │ (plan .md)  │   │ (código+DB)  │   │ (Docker)     │
  └─────────────┘   └──────────────┘   └──────┬───────┘
                                              │ OK
                                       ┌──────▼───────┐
                                       │ Deploy       │
                                       │ • bump ver   │
                                       │ • git push   │
                                       │ • scp DB     │
                                       └──────┬───────┘
                                              │
                                       ┌──────▼───────┐
                                       │ Verificar    │
                                       │ (producción) │
                                       └──────┬───────┘
                                              │ Fallo?
                                       ┌──────▼───────┐
                                       │ Nueva ver    │
                                       │ (X.Y+1)      │
                                       └──────────────┘
```
